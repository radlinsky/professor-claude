---

Atomic terms the field uses, each with a plain-words definition and a citation.
Terms that accumulate structure get promoted to a page in
[`concepts/`](concepts/) — see `.claude/course-authoring/knowledge-base.md`.

| Term | Plain-words definition | Citation | Related concepts |
|---|---|---|---|
| absolutely summable | A discrete random variable $X$ is absolutely summable if $\mathbb{E}[\lvert X \rvert] = \sum_{x} \lvert x \rvert\, p_X(x) < \infty$, guaranteeing that its expectation exists and is finite. Not all valid PMFs have a finite expectation (e.g. the PMF $p_X(k) = 6/(\pi^2 k^2)$ sums to 1 but $\mathbb{E}[X]$ diverges). For continuous random variables the analogous condition is absolute integrability. | [@chan2021probabilitydatascience, p. 130] | [expectation-and-moments](concepts/expectation-and-moments.md) |
| adjusted $R^2$ | A modification of $R^2$ that penalizes unnecessary variables: $1 - \frac{\text{RSS}/(n-d-1)}{\text{TSS}/(n-1)}$, where $d$ is the number of predictors. Unlike $R^2$ (which always increases with more variables), adjusted $R^2$ can decrease when an uninformative variable is added. A large adjusted $R^2$ indicates low test error. Despite its popularity, adjusted $R^2$ has weaker theoretical justification than AIC, BIC, or $C_p$. | [@james2021introductionstatisticallearning, pp. 234--235, Eq. 6.4] | [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| affine combination | A linear combination of vectors whose coefficients sum to one: $\beta_1 a_1 + \cdots + \beta_m a_m$ with $\beta_1 + \cdots + \beta_m = 1$. When the coefficients are additionally nonneg­ative, it is called a convex combination (or weighted average). | [@boyd2018appliedlinearalgebra, p. 17] | [vectors-and-operations](concepts/vectors-and-operations.md) |
| AIC (Akaike information criterion) | A criterion for model selection defined for models fit by maximum likelihood. For least squares with Gaussian errors, $\text{AIC} = \frac{1}{n}(\text{RSS} + 2d\hat\sigma^2)$, which is proportional to $C_p$. Penalizes model complexity (number of predictors $d$) to avoid overfitting. Select the model with the lowest AIC. | [@james2021introductionstatisticallearning, p. 234] | [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| almost surely (a.s.) | An event holds almost surely if its probability is 1 except on measure zero sets. In continuous sample spaces, events with probability 1 may exclude individual points (which have measure zero) without affecting the overall probability. | [@chan2021probabilitydatascience, p. 73] | [probability-and-odds](concepts/probability-and-odds.md) |
| alternative hypothesis | The hypothesis $H_a$ that represents something different from the default state of the world (the null hypothesis $H_0$). Typically states that an effect or difference exists. The treatment of $H_0$ and $H_a$ is asymmetric: $H_0$ is the default, and we focus on using data to reject $H_0$. Rejecting $H_0$ provides evidence in favor of $H_a$ -- a "discovery." Failing to reject $H_0$ does not prove it is true; the sample may simply be too small to detect the effect. | [@james2021introductionstatisticallearning, pp. 553--554] | [false-discovery-rate](concepts/false-discovery-rate.md) |
| AUC (area under the ROC curve) | A single-number summary of a classifier's overall performance across all classification thresholds. Equals the area under the ROC curve; ranges from 0 (perfectly wrong classifier) through 0.5 (no better than random guessing) to 1.0 (perfect classifier). A larger AUC means the classifier better discriminates between classes. Equivalent to the probability that the classifier ranks a randomly chosen positive case above a randomly chosen negative case. | [@james2021introductionstatisticallearning, p. 151] | [classification-evaluation](concepts/classification-evaluation.md) |
| average excess | The difference between the mean genotypic value of individuals carrying at least one copy of allele $B_i$ and the population mean genotypic value: $\alpha_i^* = \sum_j P_{ij\mid i} G_{ij} - \mu_G$. Under random mating the average excess equals the additive effect ($\alpha_i^* = \alpha_i$); under inbreeding (coefficient $f$), $\alpha_i = \alpha_i^*/(1+f)$. The average excess has a simple biological interpretation (conditional mean deviation), whereas the additive effect is a least-squares regression parameter. The covariance between genotypic value and allele count is $\sigma(G, N_i) = 2p_i\alpha_i^*$, linking the average excess directly to the genetic covariance. | | |
| average predictor | In the bias-variance framework, the expected prediction function across all possible training sets: $\bar{g}(\boldsymbol{x}') = \mathbb{E}_{\mathcal{D}_{\text{train}}}[g^{(\mathcal{D}_{\text{train}})}(\boldsymbol{x}')]$. The bias is the deviation of $\bar{g}$ from the true function $f$; the variance measures how individual predictors $g^{(\mathcal{D})}$ fluctuate around $\bar{g}$. For the linear model with i.i.d. Gaussian noise, $\bar{g}(\boldsymbol{x}') = f(\boldsymbol{x}')$ (the average predictor recovers the truth), so the bias is zero and all test error comes from variance and noise. | [@chan2021probabilitydatascience, pp. 433--436] | [bias-variance-trade-off](concepts/bias-variance-trade-off.md) |
| backward stepwise selection | A model selection method that begins with the full model containing all $p$ predictors and iteratively removes the least useful predictor at each step. Searches through $1 + p(p+1)/2$ models (same as forward stepwise). Requires $n > p$ so the full model can be fit, unlike forward stepwise which can be used when $p \geq n$. | [@james2021introductionstatisticallearning, pp. 231--232] | [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| bagging (bootstrap aggregation) | A general-purpose procedure for reducing the variance of a statistical learning method by training the model on $B$ bootstrap samples and averaging (regression) or majority-voting (classification) the predictions. Particularly effective for decision trees: each tree is grown deep (high variance, low bias) and the average has much lower variance. The number of trees $B$ is not a critical parameter -- increasing $B$ does not cause overfitting. | [@james2021introductionstatisticallearning, pp. 340--341] | [bagging-random-forests-and-boosting](concepts/bagging-random-forests-and-boosting.md), [decision-trees](concepts/decision-trees.md) |
| basis function (feature mapping) | A function $f_i : \mathbf{R}^n \to \mathbf{R}$ used in a linear-in-the-parameters model $\hat{f}(x) = \theta_1 f_1(x) + \cdots + \theta_p f_p(x)$. The basis functions are chosen by the modeler (e.g. $f_1(x) = 1$, $f_2(x) = x$, $f_3(x) = x^2$ for a quadratic fit); the model parameters $\theta$ are then determined by least squares. Common choices include constant, polynomial, piecewise-linear, and indicator (one-hot) functions. Choosing the basis functions is sometimes called feature engineering. | [@boyd2018appliedlinearalgebra, p. 246] | [least-squares-data-fitting](concepts/least-squares-data-fitting.md) |
| Bayes classifier | A theoretical classifier that assigns each observation to the class with the highest conditional probability $\Pr(Y = j \mid X = x_0)$. Produces the lowest possible test error rate (the Bayes error rate). In practice, the true conditional class probabilities are unknown, so the Bayes classifier serves as an unattainable gold standard against which real classifiers (such as KNN) are compared. | [@james2021introductionstatisticallearning, pp. 37--38] | [k-nearest-neighbors](concepts/k-nearest-neighbors.md), [bias-variance-trade-off](concepts/bias-variance-trade-off.md) |
| Bayes decision boundary | The set of points where two (or more) classes have equal conditional probability, i.e. where $\Pr(Y = j \mid X = x_0) = 0.5$ in a two-class problem. The Bayes classifier's prediction is determined by which side of this boundary an observation falls on. | [@james2021introductionstatisticallearning, p. 38] | [k-nearest-neighbors](concepts/k-nearest-neighbors.md) |
| Bayes error rate | The lowest achievable test error rate over all possible classifiers, equal to $1 - E\!\left(\max_j \Pr(Y = j \mid X)\right)$ averaged over all values of $X$. Analogous to the irreducible error in the regression setting: it is greater than zero whenever classes overlap in the predictor space. | [@james2021introductionstatisticallearning, pp. 38--39] | [bias-variance-trade-off](concepts/bias-variance-trade-off.md), [k-nearest-neighbors](concepts/k-nearest-neighbors.md) |
| Bayes' theorem | For events $A$ and $B$ with $\mathbb{P}[A] > 0$ and $\mathbb{P}[B] > 0$: $\mathbb{P}[A \mid B] = \mathbb{P}[B \mid A]\,\mathbb{P}[A] / \mathbb{P}[B]$. Switches the role of the conditioning: converts $\mathbb{P}[A \mid B]$ into $\mathbb{P}[B \mid A]$. The proof follows directly from writing $\mathbb{P}[A \cap B]$ in two ways via the definition of conditional probability. | [@chan2021probabilitydatascience, p. 89] | [conditional-probability](concepts/conditional-probability.md) |
| Benjamini--Hochberg procedure | A procedure for controlling the false discovery rate (FDR) at a pre-specified level $q$. Order the $m$ $p$-values $p_{(1)} \leq \cdots \leq p_{(m)}$; find $L = \max\{j : p_{(j)} < qj/m\}$; reject all hypotheses with $p_j \leq p_{(L)}$. Guarantees $\text{FDR} \leq q$ when $p$-values are independent or only mildly dependent. The rejection threshold is data-dependent (unlike Bonferroni's fixed $\alpha/m$), so the analyst cannot determine in advance whether a given $p$-value will lead to rejection. | [@james2021introductionstatisticallearning, pp. 571--572, Eq. 13.10] | [false-discovery-rate](concepts/false-discovery-rate.md) |
| best subset selection | A model selection method that fits a separate least squares regression for every possible combination of the $p$ predictors ($2^p$ models total), identifies the best model of each size by RSS, and then selects among sizes using an estimate of test error. Computationally infeasible for $p$ greater than about 40. Also suffers from statistical problems with large $p$: the huge search space increases the chance of overfitting. | [@james2021introductionstatisticallearning, pp. 227--229] | [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| BIC (Bayesian information criterion) | A model selection criterion that replaces the $2d$ penalty of $C_p$ with $\log(n) \cdot d$: $\text{BIC} = \frac{1}{n}(\text{RSS} + \log(n)\,d\,\hat\sigma^2)$. Since $\log n > 2$ for $n > 7$, BIC penalizes large models more heavily than $C_p$ or AIC, tending to select smaller models. | [@james2021introductionstatisticallearning, p. 234, Eq. 6.3] | [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| binomial coefficient | The number of ways to choose $k$ items from $n$: $\binom{n}{k} = \frac{n!}{k!(n-k)!}$. Appears in the binomial PMF as the count of distinct orderings that produce exactly $k$ successes in $n$ trials. Read "n choose k." | [@chan2021probabilitydatascience, p. 144] | [discrete-named-distributions](concepts/discrete-named-distributions.md) |
| biplot | A single figure that displays both the principal component scores (observations) and loadings (variables) together. Each observation is plotted at its score coordinates; each variable is represented by an arrow whose direction and length show its loading on the two displayed components. In R, `biplot(prcomp(...), scale = 0)` produces this plot. | [@james2021introductionstatisticallearning, pp. 498--501] | [principal-component-analysis](concepts/principal-component-analysis.md) |
| Bonferroni correction | A multiple testing correction that controls the family-wise error rate (FWER) at level $\alpha$ by rejecting each null hypothesis only if its $p$-value is below $\alpha/m$, where $m$ is the number of tests. Follows from the union bound: $\text{FWER} \leq m \times (\alpha/m) = \alpha$. Applies regardless of whether the $m$ tests are independent, making it very general. However, it is often conservative (the true FWER is well below $\alpha$), and Holm's step-down procedure is uniformly more powerful while controlling the same FWER. | [@james2021introductionstatisticallearning, pp. 562--563] | [false-discovery-rate](concepts/false-discovery-rate.md) |
| boosting | An ensemble method that builds trees sequentially, each fitting the residuals of the current model. Each tree is small (few splits, controlled by $d$), and its contribution is scaled by a shrinkage parameter $\lambda$. The final model is $\hat{f}(x) = \sum_{b=1}^{B} \lambda \hat{f}^b(x)$. Unlike bagging, boosting can overfit if $B$ is too large. Three tuning parameters: number of trees $B$ (via CV), shrinkage $\lambda$ (typical values 0.01--0.001), and interaction depth $d$ (number of splits per tree). | [@james2021introductionstatisticallearning, pp. 345--347, Algorithm 8.2] | [bagging-random-forests-and-boosting](concepts/bagging-random-forests-and-boosting.md), [decision-trees](concepts/decision-trees.md) |
| bootstrap | A resampling method that estimates the uncertainty (e.g. standard error) of a statistic by repeatedly drawing $n$ observations with replacement from the original data set. Each bootstrap sample $Z^{*r}$ is the same size as the original data but typically contains some observations more than once and omits others. The bootstrap standard error is the standard deviation of the statistic computed across $B$ bootstrap samples: $\text{SE}_B(\hat\alpha) = \sqrt{\frac{1}{B-1}\sum_{r=1}^{B}(\hat\alpha^{*r} - \bar{\hat\alpha}^*)^2}$. | [@james2021introductionstatisticallearning, pp. 209--212] | [resampling-bootstrap-and-permutation](concepts/resampling-bootstrap-and-permutation.md) |
| Borel sigma-field | The sigma-field on the real line generated by semi-closed intervals $(-\infty, b] = \{x \mid -\infty < x \le b\}$. All standard subsets of the real line (open, closed, half-open intervals, singletons) belong to the Borel sigma-field, making it the standard event space for continuous probability models. | [@chan2021probabilitydatascience, pp. 65--66] | [probability-and-odds](concepts/probability-and-odds.md) |
| $C_p$ (Mallow's $C_p$) | An estimate of test MSE for a least squares model with $d$ predictors: $C_p = \frac{1}{n}(\text{RSS} + 2d\hat\sigma^2)$, where $\hat\sigma^2$ is an estimate of the error variance. If $\hat\sigma^2$ is unbiased, $C_p$ is an unbiased estimate of test MSE. Select the model with the lowest $C_p$. Proportional to AIC for least squares models with Gaussian errors. | [@james2021introductionstatisticallearning, pp. 233--234, Eq. 6.2] | [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| Cauchy distribution | A continuous distribution with PDF $f_X(x) = 1/(\pi(1+x^2))$ for $x \in \mathbb{R}$. The Cauchy is the canonical example of a valid probability distribution whose expectation does not exist: $\mathbb{E}[\lvert X \rvert] = \infty$ because the tails decay too slowly. It demonstrates that having a valid PDF does not guarantee finite moments. | [@chan2021probabilitydatascience, pp. 183--184] | [expectation-and-moments](concepts/expectation-and-moments.md) |
| Cauchy--Schwarz inequality | The inequality $\lvert a^\top b \rvert \le \lVert a \rVert \, \lVert b \rVert$ for any $n$-vectors $a$ and $b$. Equality holds iff one vector is a nonneg. scalar multiple of the other (or one is zero). Guarantees that $a^\top b / (\lVert a \rVert \, \lVert b \rVert)$ lies in $[-1, 1]$, making the angle definition well-defined. Also used to prove the triangle inequality. | [@boyd2018appliedlinearalgebra, pp. 56--57] | [norm-and-distance](concepts/norm-and-distance.md) |
| CDF (cumulative distribution function) | The cumulative sum of a PMF (discrete) or the integral of a PDF (continuous): $F_X(x) = \mathbb{P}[X \le x]$. For discrete random variables it is a staircase function with jumps at states where $p_X(x) > 0$, bounded between 0 and 1. The PMF can be recovered from the CDF by differencing: $p_X(x_k) = F_X(x_k) - F_X(x_{k-1})$. The CDF is right continuous and non-decreasing. | [@chan2021probabilitydatascience, pp. 121--124] | [random-variables-and-distributions](concepts/random-variables-and-distributions.md) |
| central limit theorem (CLT) | The theorem stating that the CDF of the standardized sample average $Z_N = \sqrt{N}(\overline{X}_N - \mu)/\sigma$ converges to the standard Gaussian CDF $\Phi(z)$ as $N \to \infty$, for i.i.d. random variables with finite mean $\mu$, variance $\sigma^2$, and $\mathbb{E}[\lvert X \rvert^3] < \infty$. The CLT does NOT say the PDF converges to a Gaussian; it says only that the CDF values converge pointwise. It fails for the Cauchy distribution (no finite third moment) and is only a good approximation near the mean (not in the tails, where Chernoff's bound is needed). | [@chan2021probabilitydatascience, pp. 367--372, 380] | [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| Chebyshev's inequality | For any random variable $X$ with mean $\mu$ and variance $\sigma^2$: $\mathbb{P}[\lvert X - \mu \rvert \ge \varepsilon] \le \text{Var}[X]/\varepsilon^2$. Derived by applying Markov's inequality to $(X - \mu)^2$. In the alternative form $\varepsilon = k\sigma$: $\mathbb{P}[\lvert X - \mu \rvert \ge k\sigma] \le 1/k^2$. Applied to the sample mean: $\mathbb{P}[\lvert \overline{X}_N - \mu \rvert > \epsilon] \le \sigma^2/(N\epsilon^2)$, which directly proves the weak law of large numbers. Bounds are valid but often loose (polynomial decay $O(1/N)$). | [@chan2021probabilitydatascience, pp. 341--343] | [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| Chernoff's bound | For any random variable $X$ and $\varepsilon \ge 0$: $\mathbb{P}[X \ge \varepsilon] \le e^{-\varphi(\varepsilon)}$, where $\varphi(\varepsilon) = \max_{s>0}\{s\varepsilon - \log M_X(s)\}$ and $M_X(s)$ is the MGF. The proof uses Markov's inequality on $e^{sX}$ and optimizes over $s$. For a Gaussian with variance $\sigma^2/N$: $\mathbb{P}[X \ge \varepsilon] \le \exp\{-\varepsilon^2 N/(2\sigma^2)\}$, giving exponential decay in $N$ (much tighter than Chebyshev's $O(1/N)$). Named after Herman Chernoff, though the bound was actually discovered by his colleague Herman Rubin. | [@chan2021probabilitydatascience, pp. 343--346, 351] | [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| cluster centroid (group representative) | The mean of the vectors assigned to a group in a clustering: $z_j = (1/\lvert G_j \rvert) \sum_{i \in G_j} x_i$. Minimizes the mean squared distance from the group's vectors to the representative. The entries of the centroid give average feature values for the group, making the clustering interpretable. | [@boyd2018appliedlinearalgebra, p. 74] | [clustering-and-k-means](concepts/clustering-and-k-means.md) |
| clustering objective ($J^{\text{clust}}$) | The mean squared distance from each data vector to its assigned group representative: $J^{\text{clust}} = (\lVert x_1 - z_{c_1} \rVert^2 + \cdots + \lVert x_N - z_{c_N} \rVert^2) / N$. A smaller value means a better clustering. Finding the globally optimal clustering (assignments and representatives that minimize $J^{\text{clust}}$) is computationally intractable; the $k$-means algorithm finds a suboptimal (but typically very good) solution. | [@boyd2018appliedlinearalgebra, pp. 72--73] | [clustering-and-k-means](concepts/clustering-and-k-means.md) |
| conditional expectation | The expectation of $X$ given that $Y$ is fixed at $Y = y$: $\mathbb{E}[X \mid Y = y] = \sum_x x\, p_{X \mid Y}(x \mid y)$ (discrete) or $\int_{-\infty}^{\infty} x\, f_{X \mid Y}(x \mid y)\, dx$ (continuous). The result is a function of $y$ only (all randomness in $X$ has been averaged out). The law of total expectation connects conditional and unconditional expectations: $\mathbb{E}[X] = \mathbb{E}_Y[\mathbb{E}_{X \mid Y}[X \mid Y]]$. | [@chan2021probabilitydatascience, pp. 275--276] | [covariance-and-correlation](concepts/covariance-and-correlation.md), [joint-distributions](concepts/joint-distributions.md) |
| confidence interval | A random interval $[\hat{\Theta} - \epsilon,\; \hat{\Theta} + \epsilon]$ constructed from an estimator $\hat{\Theta}$ such that $\mathbb{P}[\hat{\Theta} - \epsilon \le \theta \le \hat{\Theta} + \epsilon] \ge 1 - \alpha$. The interval is random (it changes with each dataset); the true parameter $\theta$ is fixed. Saying "95% confidence" means that 95% of such intervals constructed from repeated experiments will contain $\theta$, not that $\theta$ has a 95% probability of lying in any particular realized interval. | [@chan2021probabilitydatascience, pp. 547--549, Eqs. 9.2--9.4] | [confidence-intervals-and-hypothesis-testing](concepts/confidence-intervals-and-hypothesis-testing.md) |
| confidence level | The probability $1 - \alpha$ that a confidence interval contains the true parameter $\theta$. Typically set to 0.95 ($\alpha = 0.05$). A higher confidence level produces a wider interval (more certainty requires a larger margin of error). | [@chan2021probabilitydatascience, p. 549, Eq. 9.4] | [confidence-intervals-and-hypothesis-testing](concepts/confidence-intervals-and-hypothesis-testing.md) |
| confounding | A phenomenon in which the marginal relationship between a predictor and the response differs from the conditional relationship (after accounting for other predictors). In classification, a predictor can appear positively associated with the response in a simple regression but negatively associated in a multiple regression because it is correlated with another predictor. The phenomenon is the same as in linear regression and is a serious risk when performing single-predictor regressions while other relevant predictors are available. | [@james2021introductionstatisticallearning, pp. 138--140] | [classification-evaluation](concepts/classification-evaluation.md) |
| confusion matrix | A table that compares predicted class labels against true class labels for a classifier. In a two-class problem, the table has four cells: true positives (TP), true negatives (TN), false positives (FP), and false negatives (FN). Reveals the distribution of errors across classes, which overall error rate alone does not capture. | [@james2021introductionstatisticallearning, pp. 148--149] | [classification-evaluation](concepts/classification-evaluation.md) |
| conjugate prior | A prior distribution $f_\Theta(\theta)$ such that the posterior $f_{\Theta \mid \boldsymbol{X}}(\theta \mid \boldsymbol{x})$ has the same functional form as the prior after multiplying by the likelihood. Many common likelihood families (especially exponential-family distributions) have conjugate priors, though not every likelihood family does. Key pairs: Beta for Bernoulli, Gaussian for Gaussian mean, Gamma for Poisson rate, Inverse Gamma for Gaussian variance. Conjugate priors are computationally convenient but not necessarily the best modeling choice. | [@chan2021probabilitydatascience, pp. 515--518, Table 8.1] | [maximum-a-posteriori-estimation](concepts/maximum-a-posteriori-estimation.md) |
| consistent estimator | An estimator $\hat{\Theta}_N$ is consistent if $\hat{\Theta}_N \xrightarrow{p} \theta$ as $N \to \infty$, i.e., $\lim_{N \to \infty} \mathbb{P}[\lvert\hat{\Theta}_N - \theta\rvert \geq \epsilon] = 0$ for every $\epsilon > 0$. Convergence in mean squared error ($\lim \mathbb{E}[(\hat{\Theta}_N - \theta)^2] = 0$) is sufficient for consistency via Chebyshev's inequality, but the converse does not hold. Consistency and unbiasedness are independent properties: $\hat{\Theta} = X_1$ is unbiased but inconsistent; the ML variance estimator $(1/N)\sum(X_n - \hat{\mu})^2$ is biased but consistent. Under regularity conditions, ML estimators of i.i.d. data are consistent. | [@chan2021probabilitydatascience, pp. 494--499, Def. 8.6] | [likelihood-and-log-likelihood](concepts/likelihood-and-log-likelihood.md) |
| convergence in distribution | A sequence of random variables $Z_1, Z_2, \ldots$ converges in distribution to $Z$ if $\lim_{N \to \infty} F_{Z_N}(z) = F_Z(z)$ for every continuous point $z$ of $F_Z$. Written $Z_N \xrightarrow{d} Z$. This is the weakest form of convergence: it says only that CDF values converge, not that the random variables themselves become close. A binomial CDF converges to a Gaussian CDF (CLT), but the binomial PDF never becomes a Gaussian PDF. | [@chan2021probabilitydatascience, pp. 368--371] | [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| convergence in probability | A sequence $A_1, A_2, \ldots$ converges in probability to $\alpha$ if for every $\varepsilon > 0$: $\lim_{N \to \infty} \mathbb{P}[\lvert A_N - \alpha \rvert > \varepsilon] = 0$. Written $A_N \xrightarrow{p} \alpha$. The limit is outside the probability (a sequence of real numbers converging to 0). Stronger than convergence in distribution but weaker than almost sure convergence. The weak law of large numbers is convergence in probability of $\overline{X}_N$ to $\mu$. | [@chan2021probabilitydatascience, pp. 356--357] | [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| convex combination | A linear combination of vectors whose coefficients are all nonnegative and sum to one; equivalently, a weighted average. A special case of an affine combination (which requires the sum-to-one constraint but allows negative weights). | [@boyd2018appliedlinearalgebra, p. 17] | [vectors-and-operations](concepts/vectors-and-operations.md) |
| convolution (probability) | The operation that gives the PDF of the sum $Z = X + Y$ of two independent random variables: $f_Z(z) = (f_X * f_Y)(z) = \int_{-\infty}^{\infty} f_X(z-y) f_Y(y)\, dy$. Intuitively, flip one PDF, slide it past the other, and integrate the overlap at each position. Convolving two uniform distributions produces a triangle; repeated convolution approaches a Gaussian (the Central Limit Theorem). | [@chan2021probabilitydatascience, pp. 281--282] | [joint-distributions](concepts/joint-distributions.md), [continuous-named-distributions](concepts/continuous-named-distributions.md) |
| correlation coefficient | The cosine of the angle between the de-meaned versions of two vectors: $\rho = \tilde{a}^\top \tilde{b} / (\lVert \tilde{a} \rVert \, \lVert \tilde{b} \rVert)$, where $\tilde{a} = a - \mathrm{avg}(a)\mathbf{1}$. Ranges from $-1$ (anti-aligned de-meaned vectors) to $+1$ (aligned). When $\rho = 0$ the vectors are uncorrelated. Sometimes expressed as a percentage. | [@boyd2018appliedlinearalgebra, p. 60] | [norm-and-distance](concepts/norm-and-distance.md) |
| correlation-based distance | A dissimilarity measure for clustering that considers two observations similar if their feature vectors are highly correlated (similar profiles), regardless of their Euclidean distance. Useful when the pattern of features matters more than their magnitude (e.g., grouping shoppers by purchase preferences rather than volume). | [@james2021introductionstatisticallearning, pp. 525--527] | [clustering-and-k-means](concepts/clustering-and-k-means.md) |
| cost complexity pruning | A method for pruning decision trees that considers a sequence of subtrees indexed by a nonnegative tuning parameter $\alpha$. For each $\alpha$, find the subtree $T$ minimizing $\sum_{m=1}^{\lvert T \rvert} \sum_{i: x_i \in R_m} (y_i - \hat{y}_{R_m})^2 + \alpha \lvert T \rvert$. When $\alpha = 0$ the full tree is optimal; as $\alpha$ increases, smaller subtrees are favored. The value of $\alpha$ is selected by cross-validation. Also called weakest link pruning. Analogous to the lasso penalty for linear models. | [@james2021introductionstatisticallearning, pp. 332--333, Eq. 8.4] | [decision-trees](concepts/decision-trees.md) |
| covariance matrix | For a random vector $\boldsymbol{X} = [X_1, \ldots, X_N]^T$, the $N \times N$ matrix $\boldsymbol{\Sigma} = \text{Cov}(\boldsymbol{X}) = \mathbb{E}[(\boldsymbol{X} - \boldsymbol{\mu})(\boldsymbol{X} - \boldsymbol{\mu})^T]$, with diagonal entries $\text{Var}[X_i]$ and off-diagonal entries $\text{Cov}(X_i, X_j)$. Always symmetric and positive semi-definite. If the coordinates are independent, $\boldsymbol{\Sigma}$ is diagonal. | Hoeffding's inequality | For i.i.d. random variables $X_1, \ldots, X_N$ with $0 \le X_n \le 1$ and $\mathbb{E}[X_n] = \mu$: $\mathbb{P}[\lvert \overline{X}_N - \mu \rvert > \epsilon] \le 2e^{-2\epsilon^2 N}$. | Holm's method | A step-down procedure for controlling the FWER that is uniformly more powerful than the Bonferroni correction. | lasso | A shrinkage method that estimates regression coefficients by minimizing $\text{RSS} + \lambda \sum_{j=1}^{p} \lvert \beta_j \rvert$, where the $\ell_1$ penalty forces some coefficients to exactly zero when $\lambda$ is sufficiently large, performing variable selection. | root-mean-square (RMS) | The RMS value of an $n$-vector $x$ is $\mathrm{rms}(x) = \lVert x \rVert / \sqrt{n}$, the square root of the average of the squared entries. Gives a single number for the "typical" magnitude of entries, comparable across vectors of different sizes. Related to the standard deviation by $\mathrm{rms}(x)^2 = \mathrm{avg}(x)^2 + \mathrm{std}(x)^2$. | [@boyd2018appliedlinearalgebra, pp. 46, 53] | [norm-and-distance](concepts/norm-and-distance.md) |
| sample average (sample statistic) | The random variable $\overline{X}_N = (1/N)\sum_{n=1}^{N} X_n$ that summarizes $N$ i.i.d. observations. It is itself random (has a PDF, mean, variance). Its mean equals the population mean ($\mathbb{E}[\overline{X}_N] = \mu$) and its variance is $\sigma^2/N$, shrinking with sample size. The law of large numbers says $\overline{X}_N \to \mu$; the central limit theorem says the CDF of the standardized version approaches the Gaussian CDF. Also called a sample statistic because it summarizes microstates using random samples. | [@chan2021probabilitydatascience, pp. 320, 351--352] | [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| sample space ($\Omega$) | The set of all possible outcomes of a random experiment. Outcomes must be exhaustive (every possibility appears) and mutually exclusive (no outcome belongs to two categories). The sample space can be finite (die: $\{1,2,3,4,5,6\}$), countably infinite (counting events), or uncountable (an interval like $[0,1]$). Together with an event space and a probability law, it forms a probability space. | [@chan2021probabilitydatascience, pp. 56--58] | [probability-and-odds](concepts/probability-and-odds.md) |
| scree plot | A plot of the proportion of variance explained (PVE) by each successive principal component, used to decide how many components to retain. The name comes from the resemblance to the scree (loose rocks) at the base of a mountain. An "elbow" in the plot -- a point where the marginal PVE drops sharply -- suggests the number of components beyond which additional components add little. | [@james2021introductionstatisticallearning, pp. 504--505, 507--508] | [principal-component-analysis](concepts/principal-component-analysis.md) |
| sensitivity (true positive rate) | The fraction of truly positive observations that a classifier correctly identifies: $\text{TP}/\text{P}$. Also called recall, power, or $1 - \text{Type II error rate}$. A classifier with high sensitivity catches most of the positive cases but may do so at the expense of specificity. In the ISL Default example, LDA with a 50% threshold has sensitivity of only 24.3% (misses 75.7% of defaulters), despite an overall error rate of only 2.75%. | [@james2021introductionstatisticallearning, pp. 149, 152--153] | [classification-evaluation](concepts/classification-evaluation.md) |
| shrinkage penalty | The term $\lambda \sum_{j=1}^{p} \beta_j^2$ (ridge) or $\lambda \sum_{j=1}^{p} \lvert \beta_j \rvert$ (lasso) added to the RSS in penalized regression. The penalty is small when the coefficients are close to zero, so it shrinks the coefficient estimates toward zero. The intercept $\beta_0$ is not penalized. The tuning parameter $\lambda$ controls how much the penalty affects the estimates: $\lambda = 0$ gives ordinary least squares; larger $\lambda$ gives more shrinkage. | [@james2021introductionstatisticallearning, pp. 237--238, 241--242] | [ridge-regularization](concepts/ridge-regularization.md) |
| sigma-field ($\sigma$-field) | An event space $\mathcal{F}$ that is closed under countably infinite unions and intersections (in addition to being a field, i.e., closed under complement, finite union, and finite intersection). Required for sample spaces with countably or uncountably many elements; for finite sample spaces, the power set already qualifies. The Borel sigma-field, generated by semi-closed intervals on the real line, is the standard choice for continuous probability models. | [@chan2021probabilitydatascience, pp. 64--66] | [probability-and-odds](concepts/probability-and-odds.md) |
| skewness | The third standardized central moment $\gamma = \mathbb{E}[((X-\mu)/\sigma)^3]$, measuring the asymmetry of a distribution. Positive skewness means the right tail is longer (distribution is right-skewed); negative skewness means the left tail is longer. A symmetric distribution (e.g. Gaussian) has skewness 0. Skewness provides shape information beyond mean and variance. | [@chan2021probabilitydatascience, p. 216] | [expectation-and-moments](concepts/expectation-and-moments.md) |
| soft-thresholding | The operation $\hat\beta^L_j = \text{sign}(y_j)(\lvert y_j \rvert - \lambda/2)_+$ that the lasso applies to coefficient estimates in the simple orthogonal case ($n = p$, diagonal $\mathbf{X}$). Translates each coefficient toward zero by a fixed amount $\lambda/2$ and sets it to exactly zero if $\lvert y_j \rvert \leq \lambda/2$. Contrasts with ridge regression's proportional shrinkage $\hat\beta^R_j = y_j/(1+\lambda)$, which never produces exact zeros. | [@james2021introductionstatisticallearning, pp. 247--248, Eq. 6.15] | [ridge-regularization](concepts/ridge-regularization.md) |
| sparse model | A regression model in which many coefficient estimates are exactly zero, so only a subset of the predictors are used. The lasso naturally produces sparse models because its $\ell_1$ penalty forces some coefficients to exactly zero. Sparse models are more interpretable than dense ones because they identify which predictors matter. Best subset selection also produces sparse models but is computationally infeasible for large $p$. | [@james2021introductionstatisticallearning, pp. 241--242, 246] | [ridge-regularization](concepts/ridge-regularization.md), [model-selection-and-information-criteria](concepts/model-selection-and-information-criteria.md) |
| sparse vector | A vector with many entries equal to zero. The sparsity pattern is the set of indices of the nonzero entries; the count of nonzero entries is denoted $\mathrm{nnz}(x)$. Unit vectors are sparse (one nonzero entry); the zero vector is the sparsest possible vector (no nonzero entries). Sparse vectors arise in word counts, customer purchases, and Boolean occurrence records. | [@boyd2018appliedlinearalgebra, p. 6] | [vectors-and-operations](concepts/vectors-and-operations.md) |
| specificity (true negative rate) | The fraction of truly negative observations that a classifier correctly identifies: $\text{TN}/\text{N}$. The false positive rate is $1 - \text{specificity} = \text{FP}/\text{N}$ (also called Type I error rate). On the ROC curve, the x-axis is $1 - \text{specificity}$. High specificity means few false alarms. | [@james2021introductionstatisticallearning, pp. 149, 152--153] | [classification-evaluation](concepts/classification-evaluation.md) |
| standard error (se) | The standard deviation of an estimator $\hat{\Theta}$: $\text{se} = \sqrt{\text{Var}[\hat{\Theta}]}$. For the sample mean of i.i.d. data with variance $\sigma^2$, $\text{se} = \sigma/\sqrt{N}$. The standard error determines the width of the confidence interval: $\text{margin of error} = z_\alpha \cdot \text{se}$. When $\sigma$ is unknown and estimated by $\hat{S}$, the resulting standard error $\hat{S}/\sqrt{N}$ leads to a $t$-distribution-based CI rather than a Gaussian-based CI. | [@chan2021probabilitydatascience, pp. 553--556, Eq. 9.7] | [confidence-intervals-and-hypothesis-testing](concepts/confidence-intervals-and-hypothesis-testing.md) |
| standardization (z-score) | The standardized version of a vector $x$ is $z = (x - \mathrm{avg}(x)\mathbf{1}) / \mathrm{std}(x)$, which has mean zero and standard deviation one. Each entry $z_i$ is the number of standard deviations by which $x_i$ deviates from the mean of $x$. Used to make values across different vectors comparable. | [@boyd2018appliedlinearalgebra, p. 56] | [norm-and-distance](concepts/norm-and-distance.md) |
| Student's $t$-distribution | A continuous distribution with heavier tails than the Gaussian, parameterized by $\nu$ degrees of freedom. Arises when the unknown population standard deviation $\sigma$ is replaced by the sample standard deviation $\hat{S}$ in the test statistic: $T = (\hat{\Theta} - \theta)/(\hat{S}/\sqrt{N}) \sim t_\nu$ with $\nu = N - 1$. As $\nu \to \infty$, the $t$-distribution converges to the standard Gaussian. Practical rule: use $t$ when $N \le 30$; the Gaussian approximation suffices when $N \ge 30$. Developed by William Gosset (publishing under the pseudonym "Student") in 1908. | [@chan2021probabilitydatascience, pp. 556--561, Def. 9.1, Thm. 9.1] | [confidence-intervals-and-hypothesis-testing](concepts/confidence-intervals-and-hypothesis-testing.md), [sample-statistics-and-convergence](concepts/sample-statistics-and-convergence.md) |
| suboptimal clustering | A clustering (set of group assignments and representatives) found by the $k$-means algorithm. Called suboptimal because the algorithm is a heuristic that cannot guarantee the globally smallest clustering objective $J^{\text{clust}}$. Different initializations may yield different suboptimal solutions; running $k$-means several times and keeping the best result is standard practice. | [@boyd2018appliedlinearalgebra, pp. 73, 76] | [clustering-and-k-means](concepts/clustering-and-k-means.md) |
| superposition (linearity) | The defining property of a linear function: $f(\alpha x + \beta y) = \alpha f(x) + \beta f(y)$ for all vectors $x, y$ and all scalars $\alpha, \beta$. Equivalently decomposed into homogeneity ($f(\alpha x) = \alpha f(x)$) and additivity ($f(x+y) = f(x) + f(y)$). Affine functions satisfy a restricted version: superposition holds when $\alpha + \beta = 1$. | [@boyd2018appliedlinearalgebra, pp. 30--32] | [linear-functions](concepts/linear-functions.md) |
| supervised learning | A class of statistical learning problems where, for each observation of the predictor measurements $x_i$, there is an associated response $y_i$. The goal is to fit a model that relates the response to the predictors, for the purpose of prediction, inference, or both. Regression and classification are the two main types of supervised learning. | [@james2021introductionstatisticallearning, §2.1.4] | [bias-variance-trade-off](concepts/bias-variance-trade-off.md), [k-nearest-neighbors](concepts/k-nearest-neighbors.md) |
| Taylor approximation (first-order) | The affine function $\hat{f}(x) = f(z) + \nabla f(z)^\top (x - z)$ that approximates a differentiable function $f : \mathbf{R}^n \to \mathbf{R}$ near a point $z$. The gradient $\nabla f(z)$ is the vector of partial derivatives. The approximation is good when $x$ is near $z$. | [@boyd2018appliedlinearalgebra, pp. 35--36] | [linear-functions](concepts/linear-functions.md) |
| test error (test MSE) | The average prediction error (MSE for regression, misclassification rate for classification) computed on observations not used to train the model. The test error is the quantity of primary interest: it measures how well the method generalizes to new data. Estimating test error from training data alone requires resampling techniques such as cross-validation. | [@james2021introductionstatisticallearning, pp. 29--30] | [bias-variance-trade-off](concepts/bias-variance-trade-off.md) |
| test statistic | A numerical summary $T$ of the data that quantifies the evidence against the null hypothesis $H_0$. The way $T$ is constructed depends on the nature of $H_0$. For a two-sample comparison of means, the test statistic is $T = (\hat\mu_t - \hat\mu_c) / (s\sqrt{1/n_t + 1/n_c})$, where $s$ is the pooled standard deviation. A large absolute value of $T$ provides evidence against $H_0$. The $p$-value transforms the test statistic into a probability by comparing it to its null distribution. | [@james2021introductionstatisticallearning, pp. 554, Eqs. 13.1--13.2] | [false-discovery-rate](concepts/false-discovery-rate.md) |
| training error (training MSE) | The mean squared error (regression) or misclassification rate (classification) computed on the same data used to fit the model. Training error generally underestimates test error, because statistical methods often directly or indirectly minimize training error. A model can have near-zero training error while performing poorly on new data (overfitting). | [@james2021introductionstatisticallearning, pp. 29--30] | [bias-variance-trade-off](concepts/bias-variance-trade-off.md) |
| training loss | The objective function $\mathcal{E}_{\text{train}}(\boldsymbol{\theta}) = \sum_{n=1}^{N} \mathcal{L}(y_n, g_{\boldsymbol{\theta}}(x_n))$ that regression minimizes to find the best model parameters $\boldsymbol{\theta}$. The loss function $\mathcal{L}$ is a design choice: common options include squared error $(y - g(x))^2$ (convex and differentiable), absolute error $\lvert y - g(x)\rvert$ (robust to outliers but not differentiable), and cross-entropy (for binary outcomes). The squared error is standard because it is both convex and differentiable, guaranteeing an efficient unique-minimum solution. | [@chan2021probabilitydatascience, pp. 390--396, Eqs. 7.1--7.4] | [least-squares-data-fitting](concepts/least-squares-data-fitting.md), [bias-variance-trade-off](concepts/bias-variance-trade-off.md) |
| tuning parameter ($\lambda$) | The nonneg. constant $\lambda$ in ridge regression or the lasso that controls the relative impact of the shrinkage penalty on the coefficient estimates. When $\lambda = 0$, the penalty has no effect and the estimates are the same as ordinary least squares; as $\lambda \to \infty$, the impact grows and all coefficients approach zero (ridge) or become exactly zero (lasso). The value of $\lambda$ is typically selected by cross-validation. | [@james2021introductionstatisticallearning, pp. 237--238, 250--251] | [ridge-regularization](concepts/ridge-regularization.md) |
| Type I error | The error of rejecting a true null hypothesis (a false positive). The Type I error rate is the probability of making this error given that $H_0$ is true. In hypothesis testing, we typically require the Type I error rate to be at most $\alpha$ (e.g. 0.05). Type I errors are generally considered more serious than Type II errors because they involve declaring a scientific finding that is incorrect. The relationship between $\alpha$ and the $p$-value is direct: rejecting $H_0$ when the $p$-value is below $\alpha$ ensures the Type I error rate is at most $\alpha$. Also known as false positive. | [@james2021introductionstatisticallearning, pp. 557--558] | [false-discovery-rate](concepts/false-discovery-rate.md) |
| Type II error | The error of failing to reject a false null hypothesis (a false negative). The power of a test is $1 - \Pr(\text{Type II error})$, i.e. the probability of correctly rejecting $H_0$ when $H_a$ is true. In practice, there is a trade-off between Type I and Type II error rates: making the Type I error small (e.g. requiring very strong evidence to reject $H_0$) increases the Type II error rate. Also known as false negative. | [@james2021introductionstatisticallearning, pp. 557--558] | [false-discovery-rate](concepts/false-discovery-rate.md) |
| unbiased estimator | An estimator $\hat{\Theta}$ is unbiased if $\mathbb{E}[\hat{\Theta}] = \theta$ (its expected value equals the true parameter). The sample mean $(1/N)\sum X_n$ is an unbiased estimator of the Gaussian mean. The ML variance estimator $(1/N)\sum(X_n - \hat{\mu})^2$ is biased (divides by $N$ instead of $N-1$); Bessel's correction $(1/(N-1))\sum(X_n - \hat{\mu})^2$ gives the unbiased version but does not maximize the likelihood. Unbiasedness and consistency are independent properties. | [@chan2021probabilitydatascience, pp. 492--494, Def. 8.5] | [likelihood-and-log-likelihood](concepts/likelihood-and-log-likelihood.md) |
| union bound | An inequality stating that for any events $A$ and $B$, $\mathbb{P}[A \cup B] \le \mathbb{P}[A] + \mathbb{P}[B]$. Follows from inclusion-exclusion since $\mathbb{P}[A \cap B] \ge 0$. Useful when the intersection $A \cap B$ is hard to evaluate; provides a simple upper bound on the probability of "either." | [@chan2021probabilitydatascience, p. 79] | [probability-and-odds](concepts/probability-and-odds.md) |
| unsupervised learning | A class of statistical learning problems where each observation has predictor measurements but no associated response variable. The goal is to discover interesting structure in the data, such as groups or clusters, rather than to predict a response. Clustering and principal components analysis are common unsupervised methods. | [@james2021introductionstatisticallearning, §2.1.4] | [k-nearest-neighbors](concepts/k-nearest-neighbors.md) |
| validation set approach | The simplest form of cross-validation: randomly divide the observations into two parts -- a training set and a validation set (hold-out set) -- fit the model on the training set, and evaluate on the validation set. Conceptually simple but has two drawbacks: (1) the estimated test error is highly variable because it depends on the random split, and (2) only a subset of observations is used for training, so the method tends to overestimate the test error rate. | [@james2021introductionstatisticallearning, pp. 198--200] | [resampling-bootstrap-and-permutation](concepts/resampling-bootstrap-and-permutation.md) |
| variable importance | A measure of each predictor's contribution in a bagged or random forest model. For regression trees, computed as the total decrease in RSS due to splits on that predictor, averaged over all $B$ trees. For classification trees, the total decrease in Gini index is used instead. A large value indicates an important predictor. Variable importance plots partially compensate for the loss of interpretability when moving from a single tree to an ensemble. | [@james2021introductionstatisticallearning, p. 343] | [bagging-random-forests-and-boosting](concepts/bagging-random-forests-and-boosting.md) |
| within-cluster variation | A measure of how much the observations within a cluster differ from each other. For $K$-means clustering, defined as $W(C_k) = (1/\lvert C_k \rvert) \sum_{i,i' \in C_k} \sum_{j=1}^{p} (x_{ij} - x_{i'j})^2$ (sum of pairwise squared Euclidean distances, divided by cluster size). Equivalently, $2 \sum_{i \in C_k} \sum_j (x_{ij} - \bar{x}_{kj})^2$ (twice the sum of squared distances to the centroid). $K$-means minimizes total within-cluster variation $\sum_{k=1}^{K} W(C_k)$. | [@james2021introductionstatisticallearning, pp. 515--517, Eqs. 12.15--12.18] | [clustering-and-k-means](concepts/clustering-and-k-means.md) |
