Least squares data fitting uses the least squares framework to find a mathematical model $\hat{f}(x) = \theta_1 f_1(x) + \cdots + \theta_p f_p(x)$ that approximates an observed relationship $y \approx f(x)$ between a feature vector $x$ and a scalar outcome $y$, by choosing the model parameters $\theta$ to minimize the sum of squared prediction errors on the data. [@boyd2018appliedlinearalgebra, ch. 13]

**Also known as:** regression (when the basis functions include the original features); curve fitting; model fitting; empirical modeling

## Definition(s)

Given $N$ data pairs $(x^{(1)}, y^{(1)}), \ldots, (x^{(N)}, y^{(N)})$ where each $x^{(i)}$ is an $n$-vector of features and $y^{(i)}$ is a scalar outcome, a *linear in the parameters model* has the form $\hat{f}(x) = \theta_1 f_1(x) + \cdots + \theta_p f_p(x)$, where $f_1, \ldots, f_p : \mathbf{R}^n \to \mathbf{R}$ are chosen *basis functions* (also called *feature mappings*) and $\theta_1, \ldots, \theta_p$ are the *model parameters*. The model is called "linear in the parameters" because, for each fixed $x$, $\hat{f}(x)$ is a linear function of the parameter vector $\theta$. [@boyd2018appliedlinearalgebra, p. 246]

The *prediction* for data point $i$ is $\hat{y}^{(i)} = \hat{f}(x^{(i)})$. The *prediction error* or *residual* for data point $i$ is $r^{(i)} = y^{(i)} - \hat{y}^{(i)}$. [@boyd2018appliedlinearalgebra, p. 246]

In matrix-vector notation, the $N \times p$ matrix $A$ with entries $A_{ij} = f_j(x^{(i)})$ gives $\hat{y}^{\mathrm{d}} = A\theta$, so the sum of squared prediction errors is $\lVert r^{\mathrm{d}} \rVert^2 = \lVert y^{\mathrm{d}} - A\theta \rVert^2$. The *least squares fit* chooses $\hat{\theta} = (A^\top A)^{-1} A^\top y^{\mathrm{d}} = A^\dagger y^{\mathrm{d}}$, provided the columns of $A$ are linearly independent. [@boyd2018appliedlinearalgebra, p. 247]

## Notation

| Symbol | How to say it | What it means |
|---|---|---|
| $x^{(i)}$ | "x superscript i" | The $n$-vector of features for data sample $i$. The superscript $(i)$ indexes data points; $x_j^{(i)}$ is the $j$th feature of example $i$. [@boyd2018appliedlinearalgebra, p. 245] |
| $y^{(i)}$ | "y superscript i" | The scalar outcome for data sample $i$. [@boyd2018appliedlinearalgebra, p. 245] |
| $\hat{f}(x)$ | "f-hat of x" | The model or prediction function. The hat indicates it is an approximation to the unknown true function $f$. [@boyd2018appliedlinearalgebra, p. 246] |
| $\theta = (\theta_1, \ldots, \theta_p)$ | "theta" | The $p$-vector of model parameters (coefficients in the linear combination of basis functions). [@boyd2018appliedlinearalgebra, p. 246] |
| $f_i(x)$ | "f sub i of x" | The $i$th basis function or feature mapping. [@boyd2018appliedlinearalgebra, p. 246] |
| $y^{\mathrm{d}}$, $\hat{y}^{\mathrm{d}}$, $r^{\mathrm{d}}$ | "y-data, y-hat-data, r-data" | The $N$-vectors of observed outcomes, predicted outcomes, and residuals over the full data set (superscript d for "data"). [@boyd2018appliedlinearalgebra, p. 246] |
| $A_{ij} = f_j(x^{(i)})$ | "the data matrix" | The $N \times p$ matrix whose $j$th column is the $j$th basis function evaluated at all $N$ data points. [@boyd2018appliedlinearalgebra, p. 247] |
| $\mathrm{rms}(r^{\mathrm{d}})$ | "RMS fitting error" | Root-mean-square of the residuals: $\lVert r^{\mathrm{d}} \rVert / \sqrt{N}$. Measures how well the model fits the data. [@boyd2018appliedlinearalgebra, p. 246] |

## Key results & derivations

- **Data fitting reduces to least squares** -- Expressing predictions as $\hat{y}^{(i)} = A_{i1}\theta_1 + \cdots + A_{ip}\theta_p$ and stacking into matrix-vector form gives $\hat{y}^{\mathrm{d}} = A\theta$, so minimizing $\lVert r^{\mathrm{d}} \rVert^2 = \lVert y^{\mathrm{d}} - A\theta \rVert^2$ is a standard least squares problem with solution $\hat{\theta} = (A^\top A)^{-1} A^\top y^{\mathrm{d}}$. [@boyd2018appliedlinearalgebra, p. 247]
- **Constant fit equals the mean** -- With $p = 1$ and $f_1(x) = 1$, the matrix $A = \mathbf{1}$ and $\hat{\theta}_1 = \mathrm{avg}(y^{\mathrm{d}})$. The RMS fitting error is $\mathrm{std}(y^{\mathrm{d}})$, the standard deviation of the data. This provides the baseline against which all richer models are compared. [@boyd2018appliedlinearalgebra, p. 248]
- **Straight-line fit** -- With basis functions $f_1(x) = 1$ and $f_2(x) = x$ (scalar $x$), the model is $\hat{f}(x) = \theta_1 + \theta_2 x$. The optimal slope is $\hat{\theta}_2 = \rho \, \mathrm{std}(y^{\mathrm{d}}) / \mathrm{std}(x^{\mathrm{d}})$ and intercept $\hat{\theta}_1 = \mathrm{avg}(y^{\mathrm{d}}) - \hat{\theta}_2 \, \mathrm{avg}(x^{\mathrm{d}})$, where $\rho$ is the correlation coefficient between $x^{\mathrm{d}}$ and $y^{\mathrm{d}}$. The RMS fitting error is $\mathrm{std}(y^{\mathrm{d}}) \sqrt{1 - \rho^2}$, a factor $\sqrt{1 - \rho^2}$ smaller than the constant fit (exercise 13.1). [@boyd2018appliedlinearalgebra, pp. 249--251, 279]
- **Symmetric form of straight-line fit** -- When $\mathrm{std}(y^{\mathrm{d}}) \neq 0$, the straight-line fit can be written as $(\hat{y} - \mathrm{avg}(y^{\mathrm{d}})) / \mathrm{std}(y^{\mathrm{d}}) = \rho \, (x - \mathrm{avg}(x^{\mathrm{d}})) / \mathrm{std}(x^{\mathrm{d}})$: the standardized prediction equals $\rho$ times the standardized feature value. [@boyd2018appliedlinearalgebra, p. 251]
- **Polynomial fit** -- With basis functions $f_i(x) = x^{i-1}$ for $i = 1, \ldots, p$, the model is a polynomial of degree at most $p - 1$. The matrix $A$ is a Vandermonde matrix with linearly independent columns provided the data contain at least $p$ distinct values. Adding basis functions can only decrease (or keep equal) the training RMS error but may increase test error through over-fitting. [@boyd2018appliedlinearalgebra, pp. 255--256]
- **Piecewise-linear fit** -- Using $p = k + 2$ basis functions $f_1(x) = 1$, $f_2(x) = x$, $f_{i+2}(x) = (x - a_i)_+$ (where $(u)_+ = \max\{u, 0\}$) gives a continuous piecewise-affine model with $k$ knot points at $a_1, \ldots, a_k$. [@boyd2018appliedlinearalgebra, p. 257]
- **Regression is a special case** -- The general regression model $\hat{y} = x^\top \beta + v$ can be put into basis-function form with $f_1(x) = 1$ and $f_i(x) = x_{i-1}$ for $i = 2, \ldots, n+1$, giving $p = n + 1$ parameters. Conversely, any linear-in-the-parameters model can be viewed as regression with transformed features $\tilde{x} = (f_2(x), \ldots, f_p(x))$. [@boyd2018appliedlinearalgebra, pp. 257--258]
- **Auto-regressive (AR) model** -- For a time series $z_1, \ldots, z_T$, the AR model with memory $M$ predicts $\hat{z}_{t+1} = \theta_1 z_t + \cdots + \theta_M z_{t-M+1}$. The parameters are fit by least squares on $N = T - M$ examples, each with feature vector $(z_{M+i-1}, \ldots, z_i)$ and outcome $z_{M+i}$. [@boyd2018appliedlinearalgebra, p. 259]
- **Log transform for relative errors** -- When $y$ is positive and varies over a large range, fitting $w = \log y$ gives a model whose errors are approximately relative (percentage) errors: a residual $|r| = 0.05$ corresponds to roughly a 5% prediction error, since $\eta = e^{|r|} - 1 \approx |r|$ for small $|r|$. [@boyd2018appliedlinearalgebra, pp. 259--260]
- **Trend and seasonal decomposition** -- A time series can be decomposed into $\hat{y}^{\mathrm{d}} = \hat{y}^{\mathrm{lin}} + \hat{y}^{\mathrm{seas}}$ where $\hat{y}^{\mathrm{lin}}$ is a linear trend and $\hat{y}^{\mathrm{seas}}$ is a periodic component with period $P$, using $P + 1$ basis functions and least squares. The residual is the de-trended, seasonally-adjusted series. [@boyd2018appliedlinearalgebra, pp. 252, 255]
- **Mean prediction error is zero** -- With the constant function among the basis functions ($f_1(x) = 1$), the average prediction error over the training data is exactly zero: $\mathrm{avg}(\hat{r}^{\mathrm{d}}) = 0$. This follows from the orthogonality principle with $z = e_1$. [@boyd2018appliedlinearalgebra, exercise 13.6, p. 280]
- **General regression framework with choice of loss** -- Chan presents regression as choosing two ingredients: a function family $g_{\boldsymbol{\theta}}(\cdot)$ (e.g., polynomials, Legendre polynomials, Fourier basis, Gaussian basis) and a loss function $\mathcal{L}(y_n, g_{\boldsymbol{\theta}}(x_n))$. The general formulation is $\hat{\boldsymbol{\theta}} = \text{argmin}_{\boldsymbol{\theta}} \sum_{n=1}^{N} \mathcal{L}(y_n, g_{\boldsymbol{\theta}}(x_n))$. Common losses include sum-squared $(y_n - g(x_n))^2$, sum-absolute $|y_n - g(x_n)|$, cross-entropy $-(y_n \log g(x_n) + (1 - y_n)\log(1 - g(x_n)))$, and perceptual loss $\max(-y_n g(x_n), 0)$. [@chan2021probabilitydatascience, pp. 391, 396, Eqs. 7.1--7.4]
- **Robust linear regression via absolute error** -- Replacing the squared error $\sum(y_n - g(x_n))^2$ with the absolute error $\sum|y_n - g(x_n)|$ makes the regression much less sensitive to outliers, because the absolute error does not amplify large deviations. The resulting problem is not differentiable at zero but can be reformulated as a linear program (LP) by introducing slack variables $u_n = |y_n - \boldsymbol{x}_n^\top\!\boldsymbol{\theta}|$ and converting each absolute value constraint into two inequality constraints. [@chan2021probabilitydatascience, pp. 413--417, Eq. 7.13]
- **Autoregressive model** -- The two-tap AR model $y_n = a y_{n-1} + b y_{n-2}$ can be written in the matrix-vector form $\boldsymbol{y} = \boldsymbol{X}\boldsymbol{\theta} + \boldsymbol{e}$ where $\boldsymbol{X}$ is a Toeplitz matrix (each column is a one-entry shifted version of the previous). The general $L$-tap version uses $L$ previous samples. Extrapolation uses the fitted model to predict future values by feeding predicted values back as inputs, but accuracy degrades over time. [@chan2021probabilitydatascience, pp. 406--408, Eq. 7.10]

## Misconceptions & learner traps

- **Lower training error does not mean a better model** -- Adding basis functions can only reduce (or keep equal) the training RMS error, but may increase the test error. A model with lower training error can perform worse on new data due to over-fitting. The test set error, not the training error, is what matters for prediction. [@boyd2018appliedlinearalgebra, p. 262]
- **Over-fitting is not detected by training error alone** -- Over-fitting occurs when the training error is much smaller than the test error. An over-fit model is too sensitive to the specific training data and does not generalize. The remedy is to keep the model simple or use regularization (ch. 15). [@boyd2018appliedlinearalgebra, p. 262]
- **"Linear model" does not mean $\hat{f}$ is a linear function of $x$** -- A model $\hat{f}(x) = \theta_1 + \theta_2 x$ is actually affine in $x$ (it has an intercept). The name "linear" refers to the model being linear in the parameters $\theta$, not in $x$. [@boyd2018appliedlinearalgebra, p. 249]
- **Columns of $A$ must be linearly independent** -- If basis functions are linearly dependent on the given data (e.g., one is a linear combination of others at all $N$ data points), the least squares formula does not apply. This means one basis function is redundant and can be removed. [@boyd2018appliedlinearalgebra, pp. 248--249]
- **Cross-validation checks the basis functions, not a single model** -- Cross-validation produces $K$ different models (one per fold). It validates the choice of basis functions (model family), not any individual set of parameter values. The $K$ models should be similar; if they are not, the model may be unreliable. [@boyd2018appliedlinearalgebra, pp. 264--265]
- **Validation assumes test data is representative of future data** -- Out-of-sample and cross-validation assume the test data are "typical" of future data. If the data distribution shifts (e.g., consumer tastes change year to year), the test error may not predict future performance. [@boyd2018appliedlinearalgebra, p. 268]

## Teaching insights & analogies

- **Constant fit as baseline** -- The simplest model ($\hat{f} = \mathrm{avg}(y^{\mathrm{d}})$) always serves as a reference: its RMS error equals $\mathrm{std}(y^{\mathrm{d}})$. Any richer model should be compared against this baseline. If a model's RMS error is close to $\mathrm{std}(y^{\mathrm{d}})$, the features are not adding much predictive power. [@boyd2018appliedlinearalgebra, p. 248]
- **Slope as ratio of standard deviations times correlation** -- The straight-line slope $\hat{\theta}_2 = \rho \, \mathrm{std}(y^{\mathrm{d}}) / \mathrm{std}(x^{\mathrm{d}})$ has a clean interpretation: the correlation $\rho$ measures the strength and sign of the linear relationship, and the standard deviation ratio converts from $x$-units to $y$-units. [@boyd2018appliedlinearalgebra, p. 251]
- **Regression to the mean** -- In a straight-line fit, the prediction $\hat{y}$ always lies (strictly) between the mean of $y^{\mathrm{d}}$ and the data-proportional prediction, because $|\rho| < 1$. For example, tall parents tend to have children who are tall but closer to average height. This phenomenon was first observed by Francis Galton. [@boyd2018appliedlinearalgebra, exercise 13.2, p. 279]
- **Asset alpha and beta** -- In finance, the straight-line fit of an individual asset's return on the market return gives the CAPM decomposition $\hat{f}(x) = (r^{\mathrm{rf}} + \alpha) + \beta(x - \mu^{\mathrm{mkt}})$, where $\alpha$ is the asset's excess return over the risk-free rate and $\beta$ measures sensitivity to the market. [@boyd2018appliedlinearalgebra, pp. 251--252]
- **Training/test error plot reveals over-fitting** -- Plotting RMS error versus model complexity (e.g., polynomial degree) for both training and test sets shows training error monotonically decreasing while test error typically decreases then increases. The divergence point signals over-fitting. [@boyd2018appliedlinearalgebra, pp. 263--264]
- **Cross-validation as stability check** -- Beyond estimating prediction error, cross-validation checks whether model coefficients are stable across folds. If they change drastically, the model may be unreliable. The RMS cross-validation error $\sqrt{(\epsilon_1^2 + \cdots + \epsilon_K^2) / K}$ summarizes expected performance on new data. [@boyd2018appliedlinearalgebra, pp. 264--265]
- **Feature engineering as art guided by validation** -- Boyd & Vandenberghe stress three rules: try simple models first; compare candidates using validation; and remember that adding features can easily lead to over-fitting. [@boyd2018appliedlinearalgebra, p. 274]
- **Feature standardization** -- Standardizing features (subtracting mean, dividing by standard deviation) to produce z-scores makes the model coefficients interpretable on a common scale. The constant feature $f_1(x) = 1$ is not standardized. [@boyd2018appliedlinearalgebra, p. 269]
- **One-hot encoding for categorical features** -- A categorical feature with $l$ values is expanded into $l - 1$ Boolean indicator features (one for each non-default value). This avoids imposing an artificial ordering on the categories. [@boyd2018appliedlinearalgebra, pp. 270--271]
- **House price prediction walkthrough** -- The Sacramento house sales example illustrates the full pipeline: a simple 2-feature regression (RMS 74.8) is extended with feature engineering (piecewise-linear area, condo indicator, ZIP code one-hot encoding) to an 8-parameter model (RMS 68.3). Cross-validation confirms the improvement (69.29 vs 75.41) and rules out over-fitting. [@boyd2018appliedlinearalgebra, pp. 258, 274--278]
- **Time series validation must avoid look-ahead** -- For time series models, random train/test splits can leak future information into training ("look-ahead"), making the model appear better than it is. Instead, the training set should be all data before some time point and the test set all data after. [@boyd2018appliedlinearalgebra, p. 266]

## How the field talks about it

In applied math and engineering, "data fitting" or "model fitting" by least squares refers to the general framework of choosing basis functions and fitting parameters to minimize squared residuals. In statistics, the same procedure is called "regression" (simple linear regression for a straight-line fit, multiple regression for multiple features, polynomial regression for polynomial basis functions). The term "feature engineering" is more common in machine learning, where it refers to the art of choosing and transforming input features. "Validation" and "cross-validation" are universal terms. The $K$-fold cross-validation procedure (typically $K = 5$ or $K = 10$) is standard practice in both statistics and machine learning for model selection. [@boyd2018appliedlinearalgebra, pp. 245--246, 257, 264, 269]
